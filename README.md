# Пойдёшь гулять со мной? 💌 (v2)

Сайт-приглашение на свидание — интерактивный сценарий с анимациями. Данные о свидании приходят тебе в Telegram.

Сценарий: конверт 💌 → вопрос (печатается по буквам) → убегающая кнопка «Нет» (эскалация) → дата/время → активность → **еда и/или напитки (по выбору активности)** → **пожелания (можно пропустить)** → открытка с обратным отсчётом + кнопка «в календарь». Конфетти и фоновая музыка.

## Что нового в v2

- **Конверт на старте** — тап раскрывает письмо, потом появляется вопрос.
- **Typewriter** — вопрос печатается по буквам.
- **Кнопка «Нет» с эскалацией** — убегает, меняет текст («точно нет?» → «ну пожалуйста 🥺» → …), кнопка «Да» растёт, «Нет» сжимается и исчезает.
- **Конфетти** — при «Да» и в финале (canvas, без библиотек).
- **Шаг «активность»** — кино / прогулка / кофе / игры / 🍷 Выпьем / 🍿 Netflix & chill.
- **Умная логика еды/напитков:**
  - Спрашиваем про **еду** только если выбрано Кино, Кофе или Netflix & chill.
  - Спрашиваем про **напитки** только если выбрано «Выпьем» или «Netflix & chill» (пиво, вино, крепкое, коктейли, игристое + «свой напиток» через всплывающее окно).
  - Можно выбрать и то, и другое — тогда оба шага по очереди. Иначе сразу финал.
- **Персонализация через ссылку:**
  - `?name=Аня` — имя в вопросе, реакции и финале.
  - `?q=Сходим в кино?` — заменяет текст вопроса.
  - Вместе: `index.html?name=Аня&q=Сходим в кино?` → «Аня, сходим в кино?». Пробелы и кириллица в URL допустимы.
- **Динамика по ответам (B):** финальная фраза и подпись собираются из её выбора — Netflix → «кино, плед и мы вдвоём 🍿❤️», Выпьем → «бокал за нас 🥂», Прогулка → «погуляем и поболтаем 🌆» и т.д. В шапке открытки — эмодзи-бейдж выбранных активностей.
- **Шаг «пожелания»** — необязательное текстовое поле (место, что захватить и т.п.). Можно пропустить.
- **Иконка вкладки** — `favicon.svg` (сердечко).
- **Открытка-финал** — карточка свидания, строки еды/напитков/пожеланий показываются только если заполнены.
- **Обратный отсчёт** — «осталось N дней ЧЧ:ММ:СС», тикает в реальном времени.
- **Кнопка «Добавить в календарь»** — скачивает `.ics`.
- **Память** — после «да» повторный заход сразу показывает открытку с отсчётом. Кнопка «Пройти заново» сбрасывает.
- **Фоновая музыка** — `music.mp3` (Yiruma — River Flows in You), зациклена, плавно появляется. Стартует при открытии конверта.
- **Звук вкл по умолчанию** (🔊 справа вверху) — глушит и музыку, и звуки кликов.

## Файлы

- `index.html`, `style.css`, `script.js` — сам сайт (статика, GitHub Pages).
- `worker.js` — прокси для Cloudflare Workers, прячет токен бота.

## Настройка

Те же 4 шага, что и в v1:

1. **Бот:** @BotFather → `/newbot` → `BOT_TOKEN`. Свой `CHAT_ID` узнай через @userinfobot.
2. **Worker:** dash.cloudflare.com → Workers & Pages → Create Worker → вставь `worker.js` → Deploy. В Settings → Variables добавь секреты `BOT_TOKEN` и `CHAT_ID`.
3. **Подключи прокси:** в `script.js` вверху впиши URL воркера в `PROXY_URL`. Там же можно поменять текст вопроса `ASK_TEXT`.
4. **GitHub Pages:** залей `index.html`, `style.css`, `script.js`, `yay.svg`, `favicon.svg`, `music.mp3` → Settings → Pages → Deploy from branch → main/root.

Если `PROXY_URL` пустой — сайт работает, данные пишутся в консоль браузера, в Telegram не уходят.

## Проверка

Открой сайт, пройди шаги. В Telegram придёт:

```
💌 Свидание подтверждено!
📅 Дата: 25.06.2026
🕒 Время: 19:00
🍽 Еда: 🍕 Пицца, 🍣 Суши
💫 Занятие: 🎬 Кино, ☕ Кофе
```

> Открытка хранится в браузере (localStorage). Если хочешь сбросить — кнопка «Пройти заново».
